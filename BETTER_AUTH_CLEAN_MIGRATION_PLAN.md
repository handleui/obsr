# Better Auth Clean Migration Plan (Parallel Phase Execution)

Last updated: 2026-03-05  
Scope: Full auth cutover from WorkOS to Better Auth with clean migration assumptions (no prod users).  
Out of scope now: CLI auth migration (`dt auth login`, device/browser flow).

## 1) Goals

- Decouple auth concerns from provider-specific code.
- Replace WorkOS with Better Auth in web + observer/API paths.
- Use Better Auth OAuth capabilities with the latest plugin surface (`genericOAuth`, and `oAuthProxy` if needed for non-stable callback domains).
- Preserve machine auth paths (API keys + bearer auth for API consumers).
- Remove WorkOS runtime/env dependency from production runtime paths.

## 2) Hard assumptions for this migration

- No existing production users/sessions to preserve.
- We can reset auth tables/state without data backfill constraints.
- We can do a hard cutover and remove compatibility code instead of dual-issuer migration.

## 4) Target architecture (post-cutover)

- New auth module/package owns all auth primitives and provider integration:
  - Better Auth server config
  - OAuth provider configuration
  - Session/bearer handling
  - API key strategy integration points
- Observer no longer verifies WorkOS JWT directly; it validates Better Auth bearer/session identity.
- GitHub identity resolution no longer calls WorkOS identity APIs.
- Web auth routes use Better Auth flow directly.
- CLI remains temporarily on existing flow (explicitly deferred).

## 7) Phase-by-phase plan

## Phase 0: Bootstrap planning + boundaries

- Lock migration contract: “No back-compat with WorkOS required for users/sessions.”
- Freeze CLI auth as deferred.
- Define ownership map:
  - Auth core package owner: Phase B
  - API middleware owner: Phase C
  - GitHub identity owner: Phase D
  - Web auth owner: Phase E

## Phase 1: Build clean auth core and decouple

- Phase A lands provider-agnostic interfaces.
- Phase B lands Better Auth package + baseline plugins:
  - `genericOAuth` (primary OAuth integration)
  - `organization`
  - `apiKey`
  - `bearer`
  - optional `jwt` if service tokens/JWKS endpoint needed
  - optional `oAuthProxy` for unstable callback environments

## Phase 2: Cut observer + web to Better Auth

- Phase C swaps middleware/session resolution.
- Phase D rewires GitHub identity/membership logic.
- Phase E migrates web auth endpoints/UI/session usage.

## Phase 3: Remove WorkOS and validate

- Phase F removes dead WorkOS pieces from runtime paths.
- Run full monorepo validation and auth test suite.
- Update docs/env examples to Better Auth defaults.

## 8) Explicitly deferred backlog (CLI)

- `apps/cli/src/lib/auth.ts` device/browser flow replacement.
- `/cli/auth` and `/api/cli/token` compatibility redesign.
- Device Authorization plugin rollout for CLI parity.
- CLI credential storage format migration.

## 9) Risks and mitigations

- Risk: GitHub token lifecycle differs from WorkOS Pipes behavior.
  - Mitigation: make GitHub OAuth account data authoritative and add token refresh handling tests.
- Risk: API key semantics drift during plugin transition.
  - Mitigation: preserve `X-Detent-Token` behavior in observer with adapter layer during cutover.
- Risk: Cloudflare worker runtime/env differences for Better Auth config.
  - Mitigation: follow Better Auth Hono + Cloudflare guidance and test in worker-like env before merge.
