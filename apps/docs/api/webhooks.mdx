---
title: Webhooks
description: GitHub and GitLab webhook integration
---

Detent receives webhooks from GitHub and GitLab to track CI status and respond to commands.

## Webhook Endpoints

| Provider | Endpoint |
|----------|----------|
| GitHub | `POST /webhooks/github` |
| GitLab | `POST /webhooks/gitlab` |

<Info>
Webhooks are automatically configured when you install the GitHub App. Manual configuration is only needed for GitLab.
</Info>

## GitHub Events

| Event | Description |
|-------|-------------|
| `installation` | App installed/uninstalled |
| `installation_repositories` | Repos added/removed |
| `repository` | Repo renamed/transferred |
| `workflow_run` | CI workflow completed |
| `check_suite` | Checks requested |
| `issue_comment` | PR comments (for `@detent` commands) |

## PR Commands

Comment on a PR to trigger actions:

| Command | Description |
|---------|-------------|
| `@detent heal` | Analyze failures and push fixes |
| `@detent heal --dry-run` | Preview fixes without pushing |
| `@detent status` | Show current error status |

## Workflow Failures

When a workflow fails, Detent automatically:
1. Fetches the workflow logs
2. Posts a summary comment on the PR
3. Offers `@detent heal` to fix issues

## GitLab

<Note>
GitLab requires manual webhook configuration with a secret token in the `X-Gitlab-Token` header.
</Note>

Supported events: `push`, `pipeline`, `merge_request`
