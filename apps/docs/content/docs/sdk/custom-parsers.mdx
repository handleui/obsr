---
title: Custom Parsers
description: Add support for any CI tool
---

The SDK supports ESLint, TypeScript, Vitest, Cargo, and golangci-lint out of the box. For other tools, register custom parsers.

## Register a Tool

Use `registerTool()` to add full support for a new tool in one call:

```ts
import { registerTool, run } from "@obsr/diagnostics"

await registerTool({
  name: "pytest",
  commandPattern: /pytest/,
  jsonFlags: ["--json-report"],
  outputSource: "stdout",
  parse: (content) => {
    const data = JSON.parse(content)
    return data.tests
      .filter(t => t.outcome === "failed")
      .map(t => ({
        message: t.longrepr,
        filePath: t.nodeid.split("::")[0],
        severity: "error"
      }))
  }
})

// Now run() auto-detects and parses pytest
const result = await run("pytest tests/")
```

This registers both command detection (for `run()`) and the parser (for `extract()`) together.

## Parser Return Type

Your `parse` function should return `Diagnostic[]`:

```ts
interface Diagnostic {
  message: string           // Required: the error/warning message
  filePath?: string         // File path
  line?: number             // Line number (1-indexed)
  column?: number           // Column number (1-indexed)
  severity?: "error" | "warning"
  ruleId?: string           // Rule or error code
  stackTrace?: string       // Full stack trace if available
  suggestions?: string[]    // Suggested fixes
  fixable?: boolean         // Whether auto-fixable
}
```

## Example: Jest

```ts
await registerTool({
  name: "jest",
  commandPattern: /jest/,
  jsonFlags: ["--json"],
  outputSource: "stdout",
  parse: (content) => {
    const json = JSON.parse(content)
    return json.testResults.flatMap(file =>
      file.assertionResults
        .filter(r => r.status === "failed")
        .map(r => ({
          message: r.failureMessages.join("\n"),
          filePath: file.name,
          severity: "error"
        }))
    )
  }
})
```

## Example: RuboCop

```ts
await registerTool({
  name: "rubocop",
  commandPattern: /rubocop/,
  jsonFlags: ["--format", "json"],
  outputSource: "stdout",
  parse: (content) => {
    const json = JSON.parse(content)
    return json.files.flatMap(file =>
      file.offenses.map(offense => ({
        message: offense.message,
        filePath: file.path,
        line: offense.location.start_line,
        column: offense.location.start_column,
        severity: offense.severity === "error" ? "error" : "warning",
        ruleId: offense.cop_name,
        fixable: offense.correctable
      }))
    )
  }
})
```

## Example: mypy

```ts
await registerTool({
  name: "mypy",
  commandPattern: /mypy/,
  jsonFlags: ["--output=json"],
  outputSource: "stdout",
  parse: (content) => {
    const json = JSON.parse(content)
    return Object.entries(json).flatMap(([file, errors]) =>
      errors.map(e => ({
        message: e.message,
        filePath: file,
        line: e.line,
        column: e.column,
        severity: e.severity === "error" ? "error" : "warning",
        ruleId: e.code
      }))
    )
  }
})
```

## Low-Level Registration

For more control, register the parser and tool config separately:

```ts
import { registerParser, registerToolConfig } from "@obsr/diagnostics"

// Parser only (for use with extract())
registerParser("custom", (content) => [...])

// Tool config (for use with run())
registerToolConfig({
  commandPattern: /custom-tool/,
  jsonFlags: ["--json"],
  outputSource: "stdout",
  parser: "custom"
})
```

## API Fallback

If you can't write a parser for a tool, use the Observer API as a fallback:

```ts
import { createParser } from "@obsr/diagnostics"

const parse = createParser({ apiKey: "your-api-key" })
const result = await parse(unknownOutput)
```

Local parsing is tried first. The API is only called if local parsing fails or returns no results.
