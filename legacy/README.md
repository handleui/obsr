# Legacy

`legacy/` holds code from the old product shape.

- `legacy/api` and `legacy/resolver` are kept for reference only.
- `legacy/types` holds the old CI and resolver contracts.
- `legacy/extract` holds the old extraction compatibility layer.
- Use them for migration context or reusable implementation patterns.
- Do not add new product work here.
- Active work should target `apps/obsr` and `apps/docs`.
