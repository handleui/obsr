Parse CI configurations and trigger AI-powered resolving for failures.

## Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/v1/parse` | Parse CI configuration file |
| `POST` | `/v1/resolve` | Request AI resolving for errors |

## Parse CI Configuration

Parse a GitHub Actions or GitLab CI configuration file and return structured workflow information.

```bash
curl -X POST https://observer.detent.sh/v1/parse \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"config_type": "github_actions", "config_content": "..."}'
```

## Request Resolving

Analyze CI failures and get AI-generated fixes.

```bash
curl -X POST https://observer.detent.sh/v1/resolve \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"project_id": "...", "error_output": "..."}'
```

Options:
- `dry_run: true` - Preview fixes without applying
- `error_type` - Hint: `lint`, `type`, `test`, `build`
- `file_context` - Include relevant source files

## CLI

Most users interact with checks via the CLI:

```bash
detent check          # Run checks for changed files
detent resolve           # Auto-resolve failures
detent resolve --dry-run # Preview fixes
```

See [CLI documentation](/docs/cli/resolve) for details.
