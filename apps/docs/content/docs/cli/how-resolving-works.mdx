When your CI fails, Detent does more than report the problem. It captures the evidence, builds a focused issue, and then tries the smallest safe fix.

## The Flow

```
CI fails → Action uploads observations → Responses extraction builds issue diagnostics → AI generates fix → You approve
```

### 1. Observation Capture

The Detent GitHub Action runs at the end of your CI job. It captures the useful evidence around a failure:

- failed-step stdout and stderr
- workflow and command context
- optional artifacts when they already exist

Raw logs are valid input. Structured reports are optional extra evidence, not the contract.

### 2. Issue Extraction

Detent compacts those observations and sends them through the OpenAI Responses API with strict structured output. That step produces issue-native diagnostics such as:

- message
- severity
- category
- source
- file path
- line and column
- evidence snippet

Those diagnostics are ranked and deduped before any fix is proposed.

### 3. AI Resolving

Your issue data goes to our resolving service. An AI agent receives:

- the ranked issue snapshot
- the strongest evidence
- related project context

The AI runs in a sandbox. It can read files, run commands, and test fixes. It tries the fix, verifies it works, then submits a patch.

### 4. You Review

Fixes appear as PR comments. You see:

- What changed
- Why it changed
- A button to apply

One click and the fix is committed to your branch.

## What It Doesn't Do

Detent fixes the small stuff. It won't:

- Refactor your architecture
- Add new features
- Fix logic bugs it can't reproduce
- Touch code outside the error location

If a fix is uncertain, it'll tell you instead of guessing.

## Autofix vs. AI Resolve

Detent has two modes:

| Mode | When | What |
|------|------|------|
| **Autofix** | Deterministic fixes (formatting, imports) | Runs instantly, no AI cost |
| **AI Resolve** | Complex errors (type mismatches, test failures) | Uses Claude, costs credits |

Autofix runs first. If it can't solve the problem, AI Resolve kicks in.
