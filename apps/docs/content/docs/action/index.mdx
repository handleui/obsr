title: GitHub Action
description: Capture CI observations and enable AI resolving in your workflows
---

The Detent action runs at the end of your CI job to capture the evidence around a failure and send it for AI-powered resolving.

## What It Does

- **Captures observations** — Collects failed-step output, workflow metadata, and optional artifacts
- **Scrubs and compacts** — Removes obvious secrets and keeps the useful evidence
- **Builds issue diagnostics** — The backend turns raw observations into issue-native diagnostics with the OpenAI Responses API
- **Enables resolving** — Powers snapshots, fix suggestions, and PR comments

## Installation

Add the action as the final step in your workflow jobs:

```yaml title=".github/workflows/ci.yml"
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm ci
      - run: npm run lint
      - run: npm test

      - uses: detent/action@v1
        with:
          token: ${{ secrets.DETENT_TOKEN }}
        if: always()
```

<Callout type="warn">
The action must run with `if: always()` to capture evidence from failed steps.
</Callout>

## Performance Impact

The action usually adds **well under 3 seconds** to your job runtime:

| Project Size | Errors | Overhead |
|--------------|--------|----------|
| Small | ~10 | ~400ms |
| Medium | ~50 | ~700ms |
| Large | ~500 | ~2.5s |

For typical projects, this is **less than 1% of job duration**.

### What Happens

1. **Observation capture** (10-50ms) — Collects failed-step stdout, stderr, and workflow context
2. **Compaction** (20-500ms) — Keeps the useful evidence and trims noise
3. **Upload** (100-500ms) — Sends the observation bundle to Detent
4. **Issue extraction** — The backend builds structured issue diagnostics from that evidence

## Best Results

- Keep the action as the final `if: always()` step.
- Let failing commands print their normal output. Raw logs are valid input.
- If your tools also emit artifacts or reports, Detent can use those as extra evidence, but they are optional.
- Clear file paths, line numbers, stack traces, and command names lead to better fixes.

## What Gets Sent

The action sends the observation bundle needed to explain the failure:

| Field | Example | Purpose |
|-------|---------|---------|
| `repository` | `acme/app` | Identify project |
| `commitSha` | `a1b2c3d4...` | Link to code version |
| `workflowName` | `CI` | Workflow identification |
| `observations[]` | Failed step output | Raw evidence for extraction |
| `command` | `bun run check-types` | Reproduction context |
| `environment` | `ci` | Runtime context |
| `diagnostics[]` | Derived server-side | Ranked issue evidence |

Detent can work from raw logs. Structured artifacts are helpful, but they are not required.

### Derived Diagnostic Example

```json
{
  "message": "Type 'string' is not assignable to type 'number'.",
  "severity": "error",
  "category": "type-check",
  "source": "typescript",
  "ruleId": "TS2322",
  "filePath": "src/utils.ts",
  "line": 42,
  "column": 5,
  "evidence": "src/utils.ts:42:5 - error TS2322: Type 'string' is not assignable to type 'number'."
}
```

## Configuration

### Required Inputs

| Input | Description |
|-------|-------------|
| `token` | Your Observer API token (from `secrets.DETENT_TOKEN`) |

### Optional Inputs

| Input | Default | Description |
|-------|---------|-------------|
| `api-url` | `https://observer.detent.sh` | API endpoint (for self-hosted) |

### Full Example

```yaml title=".github/workflows/ci.yml"
name: CI
on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20

      - run: npm ci
      - run: npm run lint
        continue-on-error: true
      - run: npm test
        continue-on-error: true

      - uses: detent/action@v1
        with:
          token: ${{ secrets.DETENT_TOKEN }}
        if: always()
```

<Callout type="info">
Using `continue-on-error: true` on lint/test steps ensures the Detent action runs even when checks fail, which is when you need observation capture most.
</Callout>
