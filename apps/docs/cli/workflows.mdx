Detent works with GitHub Actions. Add our action to your workflow to collect errors and enable healing.

## Basic Setup

```yaml .github/workflows/ci.yml
name: CI
on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm ci
      - run: npm run lint
      - run: npm test

      - uses: detent/action@v1
        with:
          token: ${{ secrets.DETENT_TOKEN }}
        if: always()
```

## Capture Errors Properly

For Detent to parse errors, your tools need to output JSON. Update your lint/test commands:

### ESLint

```yaml
- run: npm run lint -- --format json --output-file eslint-report.json
  continue-on-error: true
```

### TypeScript

```yaml
- run: npx tsc --noEmit 2>&1 | tee tsc-output.txt
  continue-on-error: true
```

### Vitest

Add to your `vitest.config.ts`:

```typescript
export default defineConfig({
  test: {
    reporters: ['default', 'json'],
    outputFile: 'vitest.json'
  }
})
```

## Multiple Jobs

Add the action to each job that might fail:

```yaml
jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm ci
      - run: npm run lint -- --format json --output-file eslint-report.json
        continue-on-error: true
      - uses: detent/action@v1
        with:
          token: ${{ secrets.DETENT_TOKEN }}
        if: always()

  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm ci
      - run: npm test
        continue-on-error: true
      - uses: detent/action@v1
        with:
          token: ${{ secrets.DETENT_TOKEN }}
        if: always()
```

## Matrix Builds

Works with matrix strategies. Each matrix job reports its own errors:

```yaml
jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node: [18, 20, 22]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node }}
      - run: npm ci
      - run: npm test
        continue-on-error: true
      - uses: detent/action@v1
        with:
          token: ${{ secrets.DETENT_TOKEN }}
        if: always()
```
